Introdução ao HTML

HTML (HyperText Markup Language) é a linguagem de marcação padrão para criar páginas web. Descreve a estrutura de uma página web e consiste em uma série de elementos.

O que é um elemento HTML?

Um elemento HTML é definido por uma tag de início, algum conteúdo e uma tag de fim:

<nomedatag>Conteúdo vem aqui...</nomedatag>

Importante: HTML não é uma linguagem de programação, é uma linguagem de marcação que define a estrutura do seu conteúdo.

Estrutura Básica de um Documento HTML

<!DOCTYPE html>
<html>
<head>
  <title>Título da Página</title>
</head>
<body>

  <h1>Meu Primeiro Cabeçalho</h1>
  <p>Meu primeiro parágrafo.</p>

</body>
</html>

História do HTML

HTML foi criado por Tim Berners-Lee em 1991. A primeira versão pública foi HTML 2.0, publicada em 1995. O HTML 4.01 foi uma versão importante publicada em 1999. A versão atual é o HTML5, publicado como recomendação oficial do W3C em outubro de 2014.

Versões do HTML

Versão Ano Características Principais
HTML 1.0 1993 Versão inicial com elementos básicos
HTML 2.0 1995 Primeira versão padronizada oficialmente
HTML 3.2 1997 Suporte a tabelas, applets e texto ao redor de imagens
HTML 4.01 1999 Separação entre estrutura e apresentação (CSS)
XHTML 2000 HTML reformulado como aplicação XML
HTML5 2014 Elementos semânticos, APIs, multimídia nativa

Navegadores Web

Os navegadores web (Chrome, Edge, Firefox, Safari) leem documentos HTML e os exibem corretamente. O trabalho do navegador é transformar tags HTML em elementos visuais na tela.

Elementos HTML Básicos

Os elementos HTML são os blocos de construção de uma página web. Abaixo estão alguns dos elementos mais importantes:

Cabeçalhos

HTML possui seis níveis de cabeçalhos, de <h1> (mais importante) a <h6> (menos importante):

Exemplo:

<h1>Cabeçalho 1</h1>
<h2>Cabeçalho 2</h2>
<h3>Cabeçalho 3</h3>
<h4>Cabeçalho 4</h4>
<h5>Cabeçalho 5</h5>
<h6>Cabeçalho 6</h6>

Cabeçalho 1

Cabeçalho 2

Cabeçalho 3

Cabeçalho 4

Cabeçalho 5
Cabeçalho 6

Parágrafos

O elemento <p> define um parágrafo:

<p>Isto é um parágrafo.</p>
<p>Isto é outro parágrafo.</p>

Links

O elemento <a> define um hyperlink. O atributo mais importante é o href, que indica o destino do link:

<a href="https://www.mariogmelo.com.br">Isto é um link</a>

Imagens

O elemento <img> define uma imagem. É uma tag vazia (sem tag de fechamento) que contém apenas atributos:

<img src="imagem.jpg" alt="Texto alternativo" width="104" height="142">

Importante: Sempre use o atributo alt em imagens para acessibilidade e SEO.

Elementos de Quebra de Linha e Linha Horizontal

O elemento <br> insere uma quebra de linha, e <hr> cria uma linha horizontal:

<p>Isto é um parágrafo
com uma quebra de linha.</p>
<hr>
<p>Este é outro parágrafo após uma linha horizontal.</p>

Elementos de Formatação de Texto

HTML oferece diversos elementos para formatação de texto:

<b> - Texto em negrito
<strong> - Texto importante
<i> - Texto em itálico
<em> - Texto enfatizado
<mark> - Texto marcado
<small> - Texto pequeno
<del> - Texto deletado
<ins> - Texto inserido
<sub> - Texto subscrito
<sup> - Texto sobrescrito

Listas

HTML suporta listas ordenadas (numeradas) e não ordenadas (com marcadores):

<!-- Lista não ordenada -->
<ul>
  <li>Café</li>
  <li>Chá</li>
  <li>Leite</li>
</ul>

<!-- Lista ordenada -->
<ol>
  <li>Café</li>
  <li>Chá</li>
  <li>Leite</li>
</ol>

Divisões e Seções

O elemento <div> é um contêiner genérico usado para agrupar elementos e aplicar estilos:

<div class="container">
  <h2>Título da Seção</h2>
  <p>Texto da seção...</p>
</div>

Atributos HTML

Atributos fornecem informações adicionais sobre elementos HTML. Eles são sempre especificados na tag inicial e geralmente vêm em pares nome/valor como: name="value".

Atributos Comuns

Atributo Descrição Exemplo
id Especifica um id único para um elemento <div id="header">
class Especifica uma ou mais classes para um elemento <p class="texto-destaque">
style Especifica estilos CSS inline <h1 style="color:blue;">
src Especifica a URL para imagens <img src="imagem.jpg">
href Especifica a URL para links <a href="pagina.html">
alt Texto alternativo para imagens <img src="img.jpg" alt="Minha imagem">
title Informações extras sobre um elemento (tooltip) <p title="I'm a tooltip">Texto</p>

Atributos Globais

Estes atributos podem ser usados em qualquer elemento HTML:

Atributo Descrição Exemplo
id Especifica um id único para um elemento <div id="header">
class Especifica uma ou mais classes para um elemento <p class="texto-destaque">
style Especifica estilos CSS inline <h1 style="color:blue;">
title Informações adicionais (mostradas como tooltip) <p title="I'm a tooltip">
lang Define o idioma do conteúdo do elemento <html lang="pt-BR">
data-* Usado para armazenar dados personalizados <div data-id="123">

Atributos Booleanos

Alguns atributos não precisam de um valor, são chamados de atributos booleanos. Sua presença no elemento indica valor verdadeiro:

<input type="text" disabled>
<input type="checkbox" checked>
<input type="text" readonly>
<video controls></video>

Nota: Em HTML5, atributos booleanos podem ser escritos sem valor (apenas o nome) ou com o nome como valor (name="name"). Ambas as formas são válidas.

Atributos Globais (completos)

Alguns atributos podem ser usados em qualquer elemento HTML:

  • accesskey - Especifica uma tecla de atalho para ativar/focar um elemento
  • contenteditable - Especifica se o conteúdo do elemento é editável ou não
  • data-* - Usado para armazenar dados personalizados privados para a página
  • dir - Especifica a direção do texto para o conteúdo de um elemento
  • hidden - Especifica que um elemento não é ainda, ou não é mais, relevante
  • lang - Especifica o idioma do conteúdo do elemento
  • spellcheck - Especifica se o elemento deve ter sua ortografia e gramática verificada
  • tabindex - Especifica a orden de tabulação de um elemento
  • translate - Especifica se o conteúdo do elemento deve ser traduzido ou não

Diferença entre Elementos Semânticos e de Apresentação

Elementos como <b> e <i> definem apenas a aparência (negrito e itálico), enquanto <strong> e <em> indicam que o texto tem importância ou ênfase semântica.

Recomendação: Prefira usar elementos semânticos (<strong>, <em>) em vez de elementos de apresentação (<b>, <i>) para melhor acessibilidade e SEO.

  Texto <mark>marcado</mark> e texto <small>pequeno</small>.
  Texto <del>deletado</del> e texto <ins>inserido</ins>.
  Texto <sub>subscrito</sub> e texto <sup>sobrescrito</sup>.
</p>

Elementos Semânticos HTML

Elementos semânticos descrevem claramente seu significado tanto para o navegador quanto para o desenvolvedor.

Por que usar semântica?

  • Melhor acessibilidade - Leitores de tela podem interpretar melhor a estrutura da página
  • Melhor SEO - Mecanismos de busca entendem melhor o conteúdo da página
  • Manutenção mais fácil - Código mais legível e compreensível
  • Maior consistência - Padronização na estrutura do documento

Elementos Semânticos Comuns

Elemento Descrição
<header> Cabeçalho para um documento ou seção
<nav> Container para links de navegação
<main> Conteúdo principal de um documento
<section> Seção em um documento
<article> Conteúdo independente e autocontido
<aside> Conteúdo à parte do conteúdo principal
<footer> Rodapé para um documento ou seção
<figure> Container para conteúdo ilustrado
<figcaption> Legenda para um elemento <figure>
<mark> Texto marcado/realçado
<time> Define data/hora
<details> Detalhes adicionais que o usuário pode visualizar ou ocultar
<summary> Legenda visível para um elemento <details>

Listas HTML

HTML oferece três tipos de listas: ordenadas, não ordenadas e de descrição.

Listas Não Ordenadas

Uma lista não ordenada começa com a tag <ul>. Cada item da lista começa com a tag <li>.

Exemplo:

<ul>
  <li>Café</li>
  <li>Chá</li>
  <li>Leite</li>
</ul>
  • Café
  • Chá
  • Leite

Listas Ordenadas

Uma lista ordenada começa com a tag <ol>. Cada item da lista começa com a tag <li>.

Exemplo:

<ol>
  <li>Café</li>
  <li>Chá</li>
  <li>Leite</li>
</ol>
  1. Café
  2. Chá
  3. Leite

Listas de Descrição

Uma lista de descrição é uma lista de termos, com uma descrição para cada termo. A tag <dl> define a lista, a tag <dt> define o termo e a tag <dd> descreve o termo.

Exemplo:

<dl>
  <dt>Café</dt>
  <dd>- Bebida quente preta</dd>
  <dt>Leite</dt>
  <dd>- Bebida fria branca</dd>
</dl>
Café
- Bebida quente preta
Leite
- Bebida fria branca

Listas Aninhadas

Listas podem ser aninhadas (listas dentro de listas):

Exemplo:

<ul>
  <li>Café</li>
  <li>Chá
    <ul>
      <li>Chá preto</li>
      <li>Chá verde</li>
    </ul>
  </li>
  <li>Leite</li>
</ul>
  • Café
  • Chá
    • Chá preto
    • Chá verde
  • Leite

Atributos para Listas

Atributo Descrição Exemplo
type Especifica o tipo de marcador (para <ul>) ou numeração (para <ol>) <ul type="square">
start Especifica o valor inicial de uma lista ordenada <ol start="50">
reversed Especifica que a ordem da lista deve ser reversa (apenas <ol>) <ol reversed>

Tipos de Marcadores para Listas Não Ordenadas

  • disc - Marcador circular preto (padrão)
  • circle - Marcador circular vazio
  • square - Marcador quadrado preto
  • none - Sem marcador

Tipos de Numeração para Listas Ordenadas

  • 1 - Números (padrão)
  • A - Letras maiúsculas
  • a - Letras minúsculas
  • I - Algarismos romanos maiúsculos
  • i - Algarismos romanos minúsculos

Nota: Embora o atributo type ainda seja suportado em HTML5, recomenda-se usar CSS para estilizar listas, pois o atributo type está obsoleto em HTML5.

Tabelas HTML

Tabelas HTML permitem organizar dados em linhas e colunas.

Tabela Básica

Uma tabela em HTML consiste em células de tabela dentro de linhas e colunas.

Exemplo:

<table>
  <tr>
    <th>Nome</th>
    <th>Idade</th>
    <th>Cidade</th>
  </tr>
  <tr>
    <td>João</td>
    <td>25</td>
    <td>São Paulo</td>
  </tr>
  <tr>
    <td>Maria</td>
    <td>32</td>
    <td>Rio de Janeiro</td>
  </tr>
</table>
Nome Idade Cidade
João 25 São Paulo
Maria 32 Rio de Janeiro

Elementos de Tabela

Tag Descrição
<table> Define uma tabela
<tr> Define uma linha na tabela
<th> Define um cabeçalho de tabela
<td> Define uma célula de tabela
<caption> Define uma legenda para a tabela
<thead> Agrupa o conteúdo do cabeçalho em uma tabela
<tbody> Agrupa o conteúdo do corpo em uma tabela
<tfoot> Agrupa o conteúdo do rodapé em uma tabela
<col> Especifica propriedades da coluna para cada coluna
<colgroup> Especifica um grupo de uma ou mais colunas em uma tabela

Atributos de Tabela

Atributo Descrição
colspan Especifica o número de colunas que uma célula deve ocupar
rowspan Especifica o número de linhas que uma célula deve ocupar
scope Especifica se uma célula de cabeçalho é um cabeçalho para uma coluna, linha ou grupo

Formulários HTML

Formulários HTML são usados para coletar entrada do usuário. A entrada do usuário é geralmente enviada para um servidor para processamento.

Elemento <form>

O elemento <form> é um contêiner para diferentes tipos de elementos de entrada, como: campos de texto, caixas de seleção, botões de rádio, botões de envio, etc.

<form action="/action_page.php" method="post">
  <-- elementos do formulário -->
</form>

Elementos de Entrada

O elemento <input> é o elemento de formulário mais importante. Pode ser exibido de várias maneiras, dependendo do atributo type.

Multimídia em HTML

HTML5 oferece suporte nativo para multimídia, como áudio, vídeo e gráficos vetoriais.

Elemento <video>

O elemento <video> é usado para incorporar conteúdo de vídeo em um documento.

<video width="320" height="240" controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.ogg" type="video/ogg">
  Seu navegador não suporta o elemento de vídeo.
</video>

Elemento <audio>

O elemento <audio> é usado para incorporar conteúdo de som em um documento.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Seu navegador não suporta o elemento de áudio.
</audio>

Elemento <picture>

O elemento <picture> permite especificar múltiplas fontes de imagem para diferentes situações.

<picture>
  <source media="(min-width: 650px)" srcset="img_pink_flowers.jpg">
  <source media="(min-width: 465px)" srcset="img_white_flower.jpg">
  <img src="img_orange_flowers.jpg" alt="Flowers" style="width:auto;">
</picture>

Formatos de Mídia Suportados

Formato Extensão Descrição
MP4 .mp4 Formato de vídeo mais comum (H.264 + AAC)
WebM .webm Formato de vídeo aberto (VP8/VP9 + Vorbis/Opus)
Ogg .ogg Formato de vídeo aberto (Theora + Vorbis)
MP3 .mp3 Formato de áudio mais comum
WAV .wav Formato de áudio não comprimido
Ogg .ogg Formato de áudio aberto (Vorbis)

Atributo Valor Descrição

loop loop Faz o áudio/vídeo recomeçar automaticamente quando terminar
muted muted Define que a saída de áudio deve ser silenciada

Incorporando Conteúdo Externo

HTML também permite incorporar conteúdo de outras fontes usando elementos como <iframe>:

<!-- Incorporar um vídeo do YouTube -->
<iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

Nota: Sempre forneça conteúdo alternativo para elementos de mídia para garantir acessibilidade e compatibilidade com navegadores mais antigos.

Gráficos em HTML

HTML5 introduziu elementos para desenhar gráficos na web, como <canvas> para gráficos bitmap e <svg> para gráficos vetoriais.

Elemento <canvas>

O elemento <canvas> é usado para desenhar gráficos, via script (geralmente JavaScript).

<canvas id="myCanvas" width="200" height="100" style="border:1px solid #000000;">
  Seu navegador não suporta o elemento canvas.
</canvas>

<script>
  var c = document.getElementById("myCanvas");
  var ctx = c.getContext("2d");
  ctx.fillStyle = "#FF0000";
  ctx.fillRect(0, 0, 150, 75);
</script>

Elemento <svg>

SVG (Scalable Vector Graphics) define gráficos baseados em vetor para a Web usando XML.

<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

Diferenças entre Canvas e SVG

Canvas SVG
Baseado em pixels (bitmap) Baseado em vetores (XML)
Modificado apenas através de script Pode ser modificado através de script e CSS
Desempenho melhor para superfícies grandes Melhor desempenho com objetos complexos
Não suporta eventos Suporta eventos
Baixa escalabilidade Alta escalabilidade (sem perda de qualidade)
Ideal para jogos e manipulação de pixels Ideal para aplicações com interatividade e animações

Exemplos de Uso do Canvas

<!-- Desenhando um retângulo -->
<canvas id="myCanvas1" width="200" height="100"></canvas>
<script>
  var canvas = document.getElementById('myCanvas1');
  var context = canvas.getContext('2d');
  context.fillStyle = 'red';
  context.fillRect(10, 10, 50, 50);
</script>

<!-- Desenhando um círculo -->
<canvas id="myCanvas2" width="200" height="100"></canvas>
<script>
  var canvas = document.getElementById('myCanvas2');
  var context = canvas.getContext('2d');
  context.beginPath();
  context.arc(50, 50, 30, 0, 2 * Math.PI);
  context.stroke();
</script>

Exemplos de Uso do SVG

<!-- Retângulo -->
<svg width="400" height="100">
  <rect width="300" height="100" style="fill:rgb(0,0,255);stroke-width:3;stroke:rgb(0,0,0)" />
</svg>

<!-- Círculo -->
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
</svg>

<!-- Texto -->
<svg height="30" width="200">
  <text x="0" y="15" fill="red">Texto em SVG!</text>
</svg>

Dica: Use Canvas para manipulação intensiva de pixels (como jogos) e SVG para gráficos escaláveis que precisam ser animados ou interativos.

Exemplo de Geolocation API

<button onclick="getLocation()">Obter Localização</button>
<p id="demo"></p>

<script>
  function getLocation() {
    if (navigator.geolocation) {
      navigator.geolocation.getCurrentPosition(showPosition);
    } else {
      document.getElementById("demo").innerHTML = "Geolocation is not supported by this browser.";
    }
  }

  function showPosition(position) {
    document.getElementById("demo").innerHTML = "Latitude: " + position.coords.latitude +
    "<br>Longitude: " + position.coords.longitude;
  }
</script>

Exemplo de Web Storage API

<script>
  // Armazenar dados
  localStorage.setItem("nome", "João Silva");
  localStorage.setItem("idade", "30");

  // Recuperar dados
  var nome = localStorage.getItem("nome");
  var idade = localStorage.getItem("idade");

  // Remover item
  localStorage.removeItem("idade");

  // Limpar todo o storage
  // localStorage.clear();
</script>

Exemplo de Drag and Drop API

<div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)"></div>
<img id="drag1" src="img_logo.gif" draggable="true" ondragstart="drag(event)" width="336" height="69">

<script>
  function allowDrop(ev) {
    ev.preventDefault();
  }

  function drag(ev) {
    ev.dataTransfer.setData("text", ev.target.id);
  }

  function drop(ev) {
    ev.preventDefault();
    var data = ev.dataTransfer.getData("text");
    ev.target.appendChild(document.getElementById(data));
  }
</script>

Verificação de Suporte a APIs

Antes de usar uma API, é importante verificar se o navegador do usuário a suporta:

<script>
  // Verificar suporte para geolocation
  if (navigator.geolocation) {
    // Suporta geolocation API
  } else {
    // Não suporta geolocation API
  }

  // Verificar suporte para localStorage
  if (typeof(Storage) !== "undefined") {
    // Suporta Web Storage API
  } else {
    // Não suporta Web Storage API
  }

  // Verificar suporte para canvas
  var canvas = document.createElement('canvas');
  if (canvas.getContext) {
    // Suporta Canvas API
  } else {
    // Não suporta Canvas API
  }
</script>

Dica: Sempre verifique o suporte do navegador antes de usar APIs HTML5 e forneça fallbacks quando necessário para melhor compatibilidade.

Exercícios Práticos

Aqui estão alguns exercícios para praticar e consolidar seus conhecimentos de HTML.

Exercícios Básicos

Exercício 1: Página Pessoal

Crie uma página HTML simples sobre você mesmo. Inclua:

  • Um título com seu nome
  • Uma foto ou avatar
  • Uma breve biografia
  • Uma lista de seus hobbies
  • Informações de contato (email, redes sociais)

Exercício 2: Receita Culinária

Crie uma página com uma receita culinária. Inclua:

  • Nome da receita
  • Uma imagem do prato
  • Lista de ingredientes (lista não ordenada)
  • Modo de preparo (lista ordenada)
  • Informações adicionais (tempo de preparo, rendimento)

Exercício 3: Tabela de Preços

Crie uma tabela comparativa de preços de produtos. Inclua:

  • Nome do produto
  • Descrição
  • Preço
  • Classificação (estrelas)
  • Colspan/rowspan para agrupar categorias

Exercícios Intermediários

Exercício 4: Formulário de Contato

Crie um formulário de contato completo. Inclua:

  • Campos para nome, email e telefone
  • Menu suspenso para assunto
  • Área de texto para mensagem
  • Checkbox para newsletter
  • Botões de enviar e limpar
  • Validação básica com atributos HTML5

Exercício 5: Página de Produto

Crie uma página de detalhes de produto para uma loja online. Inclua:

  • Imagens do produto (use picture para responsividade)
  • Nome e descrição do produto
  • Preço e opções de compra
  • Características em lista
  • Avaliações de clientes
  • Produtos relacionados

Exercício 6: Artigo com Semântica

Crie um artigo longo usando elementos semânticos. Inclua:

  • Cabeçalho com título e informações do autor
  • Seções e artigos aninhados
  • Barra lateral com conteúdo relacionado
  • Rodapé com informações de copyright
  • Navegação entre seções
  • Elementos de mídia (imagens, vídeos)

Exercícios Avançados

Exercício 7: Portfólio Pessoal

Crie um site portfólio completo. Inclua:

  • Página inicial com introdução
  • Página "Sobre mim" com trajetória
  • Página de projetos com galeria
  • Página de contato com formulário
  • Navegação consistente entre páginas
  • Design responsivo (usando meta viewport)

Exercício 8: Dashboard com Tabelas

Crie um painel administrativo com dados tabulares. Inclua:

  • Tabelas com thead, tbody e tfoot
  • Diferentes tipos de dados (texto, números, datas)
  • Formulários para filtrar e buscar dados
  • Paginação de tabelas
  • Gráficos usando canvas (opcional)

Exercício 9: Aplicação de Todos com Web Storage

Crie uma aplicação de lista de tarefas (to-do) que use Web Storage. Inclua:

  • Formulário para adicionar novas tarefas
  • Lista de tarefas com opção de marcar como concluída
  • Opção de excluir tarefas
  • Filtros para mostrar todas, ativas ou concluídas
  • Persistência dos dados usando localStorage
  • Contador de tarefas pendentes

Desafios Extras

Desafio 1: Clone de Página

Escolha um site popular (Google, Wikipedia, etc.) e tente recriar sua página inicial apenas com HTML. Foque na estrutura semântica e não na estilização.

Desafio 2: Site com Acessibilidade

Crie um site aplicando todas as melhores práticas de acessibilidade: landmarks ARIA, texto alternativo, alto contraste, navegação por teclado, etc.

Desafio 3: Documentação Técnica

Crie uma página de documentação técnica para uma linguagem ou ferramenta, usando elementos semânticos e organização clara de conteúdo.

Dica: Ao realizar os exercícios, preste atenção na semântica, acessibilidade e validação do código. Use o validador do W3C para verificar seu HTML.
